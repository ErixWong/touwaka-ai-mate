## 通用设计原则

### 知识库层级结构

```
┌─────────────────────────────────────────────────────────────────────┐
│                       知识库层级架构                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Level 0: 知识库（Knowledge Base）                                   │
│  ─────────────────────────────                                      │
│  - 顶层容器，定义知识领域边界                                         │
│  - 配置：Embedding 模型、访问权限、知识库描述                          │
│                                                                     │
│  Level 1~N: 知识点树（Knowledge Point Tree）                         │
│  ─────────────────────────────                                      │
│  - 通过 parent_id 实现多层树状结构                                    │
│  - 每层可以是：分类、模块、功能、操作等                                │
│  - 叶子节点是实际的知识内容                                           │
│                                                                     │
│  跨节点: 语义关联（Knowledge Relations）                             │
│  ─────────────────────────────                                      │
│  - depends_on: 依赖关系（学习/操作前置条件）                          │
│  - related_to: 相关关系（主题相近）                                   │
│  - references: 引用关系（文档间引用）                                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 知识点类型

| 类型 | 图标 | 说明 | 示例 |
|------|------|------|------|
| 分类节点 | 📁 | 纯容器，无实际内容 | "采购模块"、"深度学习" |
| 概念说明 | 📖 | 概念/术语解释 | "什么是闭包"、"Transformer 架构" |
| 操作指南 | 🔧 | 步骤化操作说明 | "如何创建采购订单"、"如何配置环境" |
| 代码示例 | 💻 | 代码片段 + 说明 | "Python 数据处理示例" |
| 表格数据 | 📊 | 结构化数据 | "参数配置表"、"对比表" |
| 图片/流程图 | 🖼️ | 可视化内容 | "系统架构图"、"流程图" |
